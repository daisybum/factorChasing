## One-paragraph thesis
Frames crypto factor discovery as **sequential hypothesis search** ([[sequential-hypothesis-search]]):
a GPT-5.4 agent reads an append-only experiment trace, proposes *falsifiable* factor
hypotheses, and maps each to an executable recipe in a constrained point-in-time
**factor DSL** ([[constrained-factor-dsl]]). A deterministic engine enforces fixed splits,
a selection gate ([[ic-gating]]), transaction costs, and the portfolio test — the agent
**cannot change the evaluation rule within a session**. Headline: a ridge-combined
([[ridge-combination]]) equal-weight long-short portfolio reports **44.55% annualized
return, Sharpe 1.55** over 2024–2026 pure OOS after **5 bp one-way** cost.

## Verbatim anchors (for citation)
- DSL example: `rank_t(-0.6*log(1+mcap) + 0.5*MA_10(range) - 0.2*MA_3(Δvolume))`
- Gate: "IC̅ ≥ τ_IC and t_IC ≥ τ_t" (thresholds "fixed before the round and read from the experiment configuration").
- IC: "IC_t = corr_{i∈I_t}(s_{i,t}, r_{i,t+1:t+2})" (daily Pearson).
- Selection split: "Selection uses the training window only. Validation statistics may be recorded for diagnosis, but they do not determine whether a candidate enters the hold pool."
- Target: "one-day execution lag and one-day holding period"; r_{i,t+1:t+2} = P_{i,t+2}/P_{i,t+1} − 1.
- Mechanism: "small, liquidity-scarce tokens with persistent intraday range and positive trend tend to outperform."
- Failed signals: "Range changes and volume recoveries are noisy, while level-based scarcity and attention measures are more stable."
- Capacity: "The market-cap-weighted version performs poorly, indicating that the alpha is concentrated in smaller tokens and is capacity constrained." → see [[capacity-constraint]]
- Risk framing: "if an agent can freely change features, code, data splits, or evaluation rules, strong results may reflect uncontrolled search rather than valid discovery."

## Data & protocol
- Source: CoinMarketCap daily panel, **Jan 2020 – Dec 2025**.
- Universe filter: drop assets with **< 180 days** history or **below-threshold avg daily volume** → see [[survivorship-and-universe]].
- Splits: **train 2020–2022 / valid 2023 / pure OOS 2024–2026**.
  - ⚠ note: data ends Dec 2025 but OOS is labelled "2024–2026" — minor labelling slack, see [[contradictions]] §6.
- Agent loop: **5 search rounds**; per round = read trace → generate mixed candidate batch → translate to DSL → evaluate on **train** → append to log → interpret & update pools; after stopping → fit **ridge** → evaluate separately on train / valid / OOS.
